# План разработки MVP: iVision Conf TMA

**Версия:** 1.0
**Дата:** Март 2026

---

## Что входит в MVP

**Цель MVP:** запустить конференцию на собственном TMA-боте с реферальной механикой и базовым кабинетом участника. Проверить конверсию реф. программы на реальной аудитории.

**Входит в MVP:**
- Бот конференции с TMA
- Лендинг для незарегистрированных → переход на GetCourse
- Интеграция с GetCourse через Webhook
- Кабинет участника: программа + спикеры + призы/рефералка + профиль
- Реферальная механика (одноуровневая, тикеты, лидерборд)
- Веб-админка: управление программой, спикерами, настройки конференции
- Уведомления: реф. события + напоминания до конференции

**Не входит в MVP:**
- Многоуровневые реф. цепочки
- AI-генерация контента

## Этапы разработки

---

### Этап 0 — Инфраструктура и бот
**Результат:** бот зарегистрирован, сервер поднят, БД создана, TMA открывается по ссылке.

- [ ] Зарегистрировать бота в @BotFather, получить токен
- [ ] Настроить домен и HTTPS (обязательно для TMA)
- [ ] Развернуть сервер (VPS / Railway / Render)
- [ ] Создать схему БД (все таблицы из brief.md раздел 9)
- [ ] Настроить webhook бота
- [ ] Создать заглушку TMA (пустая страница, которая открывается кнопкой из бота)
- [ ] Проверить: ссылка `t.me/ivision_conf_bot?startapp=test` открывает TMA

---

### Этап 1 — Авторизация и онбординг
**Результат:** пользователь открывает TMA → система знает кто он → создаётся запись участника.

- [ ] Валидация `initData` на бэкенде (проверка подписи от Telegram)
- [ ] При первом входе: создать запись в `telegram_users` + `event_participants`
- [ ] Обработка `startapp` параметра: сохранить `ref_code` до регистрации
- [ ] Логика ветвления: незарегистрирован → лендинг, зарегистрирован → кабинет

---

### Этап 2 — Лендинг конференции
**Результат:** незарегистрированный видит красивый лендинг с CTA.

- [ ] Дизайн hero-блока (название, дата, обложка)
- [ ] Превью спикеров (3-4 карточки)
- [ ] Кнопка "Зарегистрироваться" через `MainButton` Telegram
- [ ] Кнопка открывает GetCourse landing (в браузере или WebView)
- [ ] Поддержка тёмной/светлой темы через `--tg-theme-*` переменные
- [ ] Вызов `expand()` при загрузке для полноэкранного режима
- [ ] Вызов `ready()` для предотвращения белого мигания

---

### Этап 3 — Интеграция с GetCourse
**Результат:** регистрация на GetCourse открывает полный кабинет в TMA.

- [ ] Настроить Webhook-эндпоинт: `POST /webhook/getcourse`
- [ ] GetCourse отправляет `telegram_id` + `email` при регистрации
- [ ] Бэкенд: найти участника по `telegram_id`, сменить статус `interested → registered`
- [ ] Привязать `ref_code` к зарегистрировавшемуся (начислить тикет рефереру)
- [ ] Бот присылает приветственное сообщение после регистрации
- [ ] TMA при следующем открытии показывает кабинет (не лендинг)

---

### Этап 4 — Программа конференции
**Результат:** участник видит расписание по дням. Изменение программы = одно действие в админке, всё остальное обновляется само.

**Веб-админка — ввод программы:**
- [ ] Парсер формата `ЧЧ:ММ | Тема | Имя спикера | Подарок спикера | [Трек]`
- [ ] Preview перед сохранением (таблица с разбитыми полями)
- [ ] Сохранение: upsert по `(conference_id, day, start_datetime)` — не дубли, а обновление
- [ ] При сохранении: авто-запуск генерации рассылок для изменённых сессий (см. Этап 8)
- [ ] Если сессия изменилась и есть `is_edited=true` рассылки → показать предупреждение с ссылкой на них
- [ ] Поддержка двух дней (вкладки "День 1" / "День 2")

**TMA — отображение программы:**
- [ ] Переключатель "День 1" / "День 2"
- [ ] Карточка сессии: время, название, фото спикера (миниатюра), имя, метка подарка
- [ ] Перерывы отображаются отдельным стилем (без фото)
- [ ] Цветная метка трека (если указан)
- [ ] Skeleton-loading при загрузке

---

### Этап 5 — Спикеры и публичная страница спикера
**Результат:** участник видит карточки спикеров. Каждый спикер получает ссылку на свою страницу для проверки. Информация вводится один раз — используется везде.

**Веб-админка — управление спикерами:**
- [ ] Форма: фото (загрузка в R2), афиша (загрузка), имя, slug (авто из имени), должность, компания, bio, ссылки (TG-канал, Instagram, сайт)
- [ ] Список спикеров: фото, имя, количество сессий, кнопки "редактировать" / "скрыть" / "копировать ссылку"
- [ ] Кнопка "Копировать ссылку" → URL вида `[домен]/s/[slug]` в буфер — отправить спикеру в мессенджер
- [ ] Флаг "скрыть всех спикеров" на уровне конференции

**Публичная страница спикера** (`/s/[slug]`) — без авторизации:
- [ ] Авто-сборка из БД: фото, имя, должность, компания, bio, ссылки
- [ ] Список сессий спикера: день, время, тема доклада
- [ ] Страница адаптивная (мобильная) — спикер открывает с телефона и сразу видит свою инфу
- [ ] Если данные неверны — спикер пишет организатору (автоматическая самопроверка без доступа к системе)

**TMA — список и карточка спикера:**
- [ ] Список: круглое фото, имя, должность, компания
- [ ] Детальная страница: полное bio, соцсети, список всех докладов спикера с временем
- [ ] Раздел полностью скрывается если `conference.speakers_hidden = true`

---

### Этап 6 — Реферальная программа и призы
**Результат:** участник получает реф. ссылку, приглашает друзей, видит прогресс.

- [ ] Генерация уникального `ref_code` для каждого участника при регистрации
- [ ] Отображение реф. ссылки: `t.me/ivision_conf_bot?startapp=REF_CODE`
- [ ] Кнопка "Поделиться" → `shareURL` нативный шеринг
- [ ] Кнопка "Скопировать ссылку"
- [ ] Счётчик приглашённых (в реальном времени)
- [ ] Таблица тикетов: сколько начислено и за что
- [ ] Прогресс-бар до следующего приза
- [ ] Список призов с порогами и статусом ("Получен" / "Осталось X")
- [ ] Лидерборд топ-10 по тикетам
- [ ] `HapticFeedback` при получении нового тикета

**Начисление тикетов:**
- [ ] +3 за регистрацию (при Webhook от GetCourse)
- [ ] +2 за каждого зарегистрировавшегося реферала
- [ ] +1 за заполнение профиля (email)

---

### Этап 7 — Профиль участника и заявка спикера
**Результат:** раздел "Я" с профилем и кнопкой "Стать спикером".

- [ ] Аватар и имя из Telegram (из `initData`)
- [ ] Общая статистика: тикеты, рефералов, призов получено
- [ ] Реф. ссылка с кнопкой копирования (дублирует раздел Призов)
- [ ] Кнопка "Стать спикером" → форма заявки
- [ ] Форма: тема доклада, краткое bio, контакт
- [ ] При отправке: запись в `speaker_applications` + уведомление организатору в бот

---

### Этап 8 — Авто-рассылки из программы
**Результат:** все рассылки генерируются из БД автоматически. Организатор только проверяет тексты и при необходимости правит дату или формулировку.

**Генератор рассылок (бэкенд):**
- [ ] При сохранении/изменении сессии → авто-создание записей в `broadcast_messages`:
  - `pre_5min`: scheduled_at = `session.start_datetime - 5 минут`
  - `pre_30min`: scheduled_at = `session.start_datetime - 30 минут`
  - `post_thanks`: scheduled_at = `session.start_datetime + 60 минут` (примерно после окончания)
- [ ] При старте конференции → `day_start` рассылки для каждого дня (8:00 утра каждого дня)
- [ ] Шаблонизатор: подставляет `{speaker.name}`, `{session.title}`, `{session.gift_description}`, `{track.stream_url}`
- [ ] Если сессия уже есть и её изменили:
  - Не-отредактированные рассылки (`is_edited=false`) → обновляются автоматически
  - Отредактированные (`is_edited=true`) → `needs_review=true`, организатор видит предупреждение

**Веб-админка — раздел "Рассылки":**
- [ ] Список всех рассылок с фильтром: все / запланированные / требуют проверки / отправленные
- [ ] Значок ⚠️ у рассылок с `needs_review=true`
- [ ] Редактирование: textarea текста + datetime отправки
- [ ] При редактировании: `is_edited` → true автоматически
- [ ] Кнопка "Сбросить к авто-шаблону" (восстановить авто-текст)
- [ ] Кнопка "Отправить сейчас" (для срочных)
- [ ] Кнопка "Отмена" (статус → `cancelled`)

**Планировщик (cron):**
- [ ] Каждую минуту проверяет `broadcast_messages` где `scheduled_at <= now` и `status = scheduled`
- [ ] Отправляет через Telegram Bot API всем участникам со статусом `registered`
- [ ] После отправки: `status → sent`, `sent_at → now`, `recipients_count → N`

**Реф. уведомления (триггерные, не из расписания):**
- [ ] При получении реферала: "Иван зарегистрировался по твоей ссылке! +2 тикета. У тебя теперь N тикетов."
- [ ] При достижении порога приза: сообщение с описанием приза

## Технический стек (рекомендация)

| Слой | Технология |
|---|---|
| Бэкенд | Node.js (Fastify) или Python (FastAPI) |
| БД | PostgreSQL |
| ORM | Prisma (Node) или SQLAlchemy (Python) |
| Telegram Bot | Grammy (Node) или Aiogram (Python) |
| TMA Фронтенд | React + Vite + @twa-dev/sdk |
| Веб-админка | React (тот же проект, отдельный роут /admin) |
| Хранилище медиа | Cloudflare R2 (дешевле S3) |
| Хостинг | Railway / Render / VPS |
| Планировщик | node-cron / APScheduler |
