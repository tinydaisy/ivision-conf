# Brief: iVision Conf — Telegram Mini App для онлайн-конференции

**Версия:** 3.1
**Дата:** Март 2026
**Статус:** Ready for Development

---

## 1. Суть продукта

Telegram Mini App для участников онлайн-конференции iVision. Открывается по ссылке прямо в Telegram — без скачивания приложений, без отдельной регистрации. Авторизация через Telegram-аккаунт автоматически.

**Уникальная позиция:** единственное решение на рынке СНГ, которое объединяет конференц-функционал (программа, эксперты), реферальную геймификацию, ироничный квест «Вход в элиту», B2B-витрину услуг и Telegram-нативность в одном продукте.

**Tone of Voice:** Интеллектуальный юмор. Мы не «дарим подарки за подписку», мы «проверяем на профпригодность». Весь процесс — от регистрации до доступа в чат — оформлен как ироничный квест, отсеивающий «туристов».

**Этапы квеста «iVision: Вход в элиту»** (ироничные названия для UX-копирайта):
1. **«Бюрократия»** — регистрация на GetCourse. *«Заполни данные, чтобы мы знали, куда слать счёт (шутка)»*
2. **«Круг доверия»** — подписки на каналы организатора и спикеров. *«Подпишись на экспертов. Они адекватные (в основном)»*
3. **«Допрос с пристрастием»** — заполнение анкеты. *«Нам нужно знать: ты пришёл работать или просто посмотреть на дизайн?»*
4. **«Посвящение»** — доступ в чат (900+). *«Поздравляю. Ты в элите. Вот ключ»*

## 2. Целевая аудитория

**Участник:** предприниматель, эксперт, специалист в нише iVision. Аудитория СНГ. Основной канал — Telegram.

**Организатор (внутренний пользователь):** команда iVision — управляет контентом через веб-админку.

---

## 3. Бизнес-контекст

- Конференция проводится периодически (минимум 3-4 раза в год)
- 2 дня, один последовательный поток докладов
- Регистрация участников — через GetCourse. При регистрации GetCourse передаёт в TMA через Webhook: `telegram_id`, `email`, `first_name`, `last_name`, `phone`, `getcourse_id` — эти данные сохраняются в TMA для кросс-идентификации пользователей (например, сверить участника из бота с записью в GetCourse). Для платных тарифов GetCourse не передаёт данные автоматически; решение — запустить процесс GetCourse с вебхуком на TMA (в планах)
- Интеграция: TMA интегрируется с GetCourse через Webhook: факт регистрации открывает полный кабинет в TMA
- **Ветеранство:** если `telegram_id` есть в базе участников прошлых конференций — флаг `is_veteran = true`. Ветеран пропускает этап анкеты (Level 0 уже пройден), но обязан пройти «Клубный взнос» (подписки на каналы текущей конференции) для разблокировки [Розыгрыш] и [Игра]
- Бот конференции — отдельный актив организатора
- После конференции база участников остаётся доступной для рассылок

## 4. Архитектура состояний

### Состояние 1: Незарегистрирован
Пользователь переходит по ссылке → лендинг-витрина открывается в in-app браузере.
- После регистрации на GetCourse срабатывает Webhook → пользователь возвращается в бота с доступом к разделам меню.

---

### Состояние 2: Зарегистрирован (до и во время конференции)
Bottom Tab Bar: **[Программа] [Услуги] [Розыгрыш] [Игра]**

Все 4 вкладки **всегда видны** в меню. [Программа] и [Услуги] доступны сразу. [Розыгрыш] и [Игра] видны, но при клике показывают Lock Screen до выполнения условий.

---

**[Программа]** — доступна сразу, без ограничений:
- Горизонтальный скролл карточек Хедлайнеров и Генеральных партнёров (кликабельны → профиль эксперта)
- **Информационная панель** (Dashboard) под каруселью, перед расписанием — меняется в зависимости от статуса участника:
  - Для не прошедших «Круг доверия»: кнопка «Пройти посвящение»
  - Для прошедших подписки: кнопки 💬 «Открыть чат» / «Вступить в чат» (если ещё не в группе) и ссылка на трансляцию (см. ниже)
- Расписание докладов по дням. Имена спикеров кликабельны → профиль
- **LIVE-блок** (статичный, под Dashboard, не плавающий):
  - До старта конференции: ⏳ таймер обратного отсчёта «Трансляция начнётся через X д. X ч.» — не кликабелен
  - За 10 минут до эфира и во время него: блок трансформируется в 🔴 LIVE-баннер с кнопкой «Подключиться к эфиру» — **прилипает к верху экрана (sticky) при скролле расписания**
- Upsell-баннер 💎 «Получить VIP: Записи + Все кодовые слова»

---

**[Услуги]** — доступна сразу, без ограничений:
- Витрина B2B-продуктов (наставничество, разработка, заявка на выступление)

---

**[Розыгрыш] и [Игра] — общая Lock-механика «Клубный взнос»:**

При клике на любую из этих вкладок до выполнения условий → единый **Экран-заглушка**:
- Текст: «Чтобы открыть доступ к подаркам, реферальной игре и получить первый билет — подпишитесь на каналы:»
- Список каналов: канал организатора (всегда) + каналы всех спикеров (если `require_speakers_sub = true` в настройках конференции)
- Кнопка «Проверить подписки и забрать подарки»

**После успешной проверки подписок:**
1. Начисляется **1 билет автоматически** (wow-эффект, «я только зашёл — а у меня уже есть билет!»)
2. **[Игра] и [Розыгрыш] разблокируются одновременно**
3. Pop-up: «Супер! Ты подтвердил статус 🎟 +1 билет! Розыгрыш открыт. Теперь забирай гарантированный бонус в [Игра]»
4. Редирект на [Игра]

---

**[Игра] (после разблокировки через подписки):**
- Верхняя карточка **«Уровень 0: Свои люди»** — первое, что видит пользователь
  - Кнопка «Забрать» → ссылка на GetCourse-анкету (отдельная форма, не регистрация конференции)
  - После заполнения (GetCourse Webhook) → чат-доступ + гайд
  - Pop-up: «Добро пожаловать в элиту. Вот ключ от чата 🔑»
  - Под карточкой сразу показывается: «До Уровня 1: ещё X приглашений → [Приз]» (мотивация к рефералке)
- Основной функционал: реферальная механика — прогресс-шкала подарков за друзей, реферальная ссылка, счётчик зарегистрированных по ссылке, макеты и тексты для анонсов, инструкции по анонсированию
- **Upsell-баннер НЕ показывается** в [Игра] (не отвлекаем от главного действия — приглашения людей)

---

**Онбординг при первом входе (State 2, сразу после регистрации):**
- Pop-up поверх всего интерфейса: «Заберите первый подарок и доступ в закрытый чат (900+ участников)! Для этого подпишитесь на каналы организатора и спикеров»
- Кнопка «Подписаться» → ведёт к экрану подписок (Lock Screen)
- Если закрыл Pop-up — вернётся при клике на [Розыгрыш] или [Игра]

---

### Состояние 3: После конференции
Bottom Tab Bar: **[Календарь] [Услуги] [Материалы] [Игра]**
- В заданную дату старые разделы скрываются, появляются новые
- **[Календарь]:** анонсы нетворкингов, мастер-майндов и будущих конференций
- **[Услуги]:** постоянная витрина продаж (не меняется между состояниями)
- **[Материалы]:** платные архивы прошедших конференций (оплата через GetCourse), «Главный урок» (вход в автоворонку). Платность/бесплатность каждого материала — настраивается в админке
- **[Игра]:** реферальная программа продолжает работать — люди копят баллы, приглашая коллег на события из [Календаря]

## 5. Функциональные требования

### 5.1 Лендинг (незарегистрированный)
- Отображение стороннего GetCourse-лендинга внутри мини-аппа через in-app браузер

### 5.2 Программа конференции (объединена с профилями экспертов)
- Переключение между днями (День 1 / День 2)
- Карточка доклада: время, тема, спикер (фото + имя), трек (цветовая метка, опционально) — кликабельна, ведёт на профиль эксперта
- Профили экспертов (Хедлайнеры, Коммерческие спикеры, Спикеры, Партнёры, Ген. партнёры): фото, bio, регалии, тема доклада, название подарка участникам, название подарка для розыгрыша, ссылка на TG-канал. У Партнёров и Ген. партнёров — ссылка на их продукты; тема выступления — при наличии
- Иерархия отображения: сортировка по роли + `sort_order` (настраивается в админке)
- Dashboard-панель (см. раздел 4)
- LIVE-блок с таймером / sticky-баннером (см. раздел 4)
- Upsell-баннер 💎 «Получить VIP: Записи + Все кодовые слова»
- Обновление в реальном времени

### 5.3 Розыгрыш
- Дата розыгрыша, список призов, баланс билетов (1 билет уже начислен при прохождении «Круга доверия»)
- Кнопка «Смотреть мои билеты» — список всех билетов
- Кнопка «Получить ещё билет» → объяснение про кодовое слово → поле «Ввести кодовое слово»
- Ввод кодового слова: проверяется без учёта регистра по общей базе кодов конференции. Если слово доступно и пользователь его ещё не вводил → выдаётся билет, записывается в БД. Если уже вводил или слово неверное → предупреждение с предложением попробовать снова
- Upsell-баннер 💎 «Получить VIP: Записи + Все кодовые слова»

### 5.4 Игра
- Реферальная механика (движок из проекта referalka)
- Уровень 0 «Свои люди» (анкета → чат + гайд) — см. раздел 4
- Прогресс-шкала подарков за количество приглашённых
- **Upsell-баннер НЕ показывается**

### 5.5 Услуги и Материалы
Единая таблица `commercial_items`, два типа:

**[Услуги]** — показывается в состояниях 2 и 3:
- Заявка на выступление спикером (платный отбор)
- Бесплатная консультация по нелинейным стратегиям
- Наставничество «Делай Имя и Продажи»
- Разработка TMA, автоматизации, автоворонки

**[Материалы]** — показывается только в состоянии 3:
- Архивы прошедших конференций (платно, ссылка на GetCourse)
- «Главный урок» (бесплатно, вход в воронку)
- Платность / бесплатность каждой карточки настраивается в админке

### 5.6 Уведомления (через бота)
- **Онбординг:** после регистрации — приветственное сообщение с кнопкой «Открыть приложение и пройти посвящение»
- При регистрации реферала: «Иван зарегистрировался по твоей ссылке! +1 тикет»
- Напоминания: за 3 дня, 1 день, 30 минут до конференции
- Рассылка в заданную дату — программа конференции
- **Рассылки знакомства со спикерами** (`speaker_intro`): фото + имя/фамилия + регалии + тема + название подарка + ссылка на TG-канал. По дням. Порядок: Организатор → Хедлайнер → Коммерческий → Партнёр → Спикер. Настраивается в админке
- При достижении порога призов в [Игра]
- Рассылки после конференции: материалы, итоги розыгрыша, дата розыгрыша

## 6. Веб-админка организатора

Закрытая паролем страница `[домен]/admin`.

### 6.1 Раздел «Конференция»
- Название, дата начала/окончания, статус (`draft` / `active` / `finished`)
- URL лендинга GetCourse
- Флаг `require_speakers_sub` — включить ли каналы спикеров в Lock Screen «Круга доверия» (в дополнение к каналу организатора, который всегда присутствует)
- URL оплаты VIP-тарифа (для Upsell-баннера 💎)
- Ссылки на трансляции: день 1, день 2
- Флаг `is_live` — активировать LIVE-баннер вручную (как резервный вариант помимо авто-активации за 10 минут)
- **В планах:** редактирование ироничных названий этапов квеста («Бюрократия», «Круг доверия» и т.д.) прямо из админки

### 6.2 Раздел «Программа» (Парсинг)

Формат ввода: `ЧЧ:ММ | Тема | Имя спикера | Подарок спикера | [Трек]`

```
10:00 | Открытие конференции | Марго Форбс | | Добро пожаловать!
10:30 | Маркетинг в 2026 году | Иван Петров | Чек-лист по контенту
11:15 | Перерыв
11:30 | Продажи через контент | Анна Сидорова | Шаблон скриптов продаж
```

- Вставить текстом → система парсит → показывает превью → кнопка «Сохранить»
- При повторном сохранении — существующие записи обновляются (без дублей). Спикер ищется по имени — если есть в базе, привязывается автоматически
- При сохранении: авто-генерация сессионных рассылок
- Если сессия изменилась и по ней были вручную отредактированные рассылки → предупреждение: «3 рассылки затронуты. Просмотреть?»
- Поддержка нескольких дней (вкладки «День 1» / «День 2»)

### 6.3 Раздел «Спикеры и Партнёры» (Единый реестр)

Карточки с ролями: **Организатор, Хедлайнер, Коммерческий, Спикер, Партнёр, Ген. партнёр**.

> **Коммерческий спикер** — оплатил спикерский слот. Отображается в TMA наравне со Спикером, но в рассылках знакомства и в Lock Screen с подписками идёт после Хедлайнеров.

> **Партнёры и Ген. партнёры** — могут иметь тему выступления (если выступают), ссылки на свои продукты. Ген. партнёр отмечается ролью и визуально выделяется в интерфейсе.

- Список: фото-миниатюра, имя, роль, `sort_order`
- Форма: фото (загрузка), афиша (загрузка), имя, должность, компания, bio, регалии (многострочный список), ссылки (TG-канал, Instagram, сайт, ссылка на продукты/сайт партнёра)
- **Настройка приоритета рассылок знакомства и порядка в Lock Screen:** порядок ролей (перетаскивание) и `sort_order` внутри каждой роли — задаётся здесь
- Кнопка «Копировать ссылку страницы» → `[домен]/s/[slug]` (публичная страница без логина)
- Под каждым спикером — список его сессий; клик → редактирование сессии

### 6.4 Раздел «Рассылки»
> Главная экономия времени. Тексты генерируются автоматически. Организатор только проверяет и правит при необходимости.

**Список всех рассылок** (авто-сгенерированных + ручных):
- Фильтр: предстоящие / отправленные / требуют проверки
- Каждая строка: тип, дата отправки, первые 80 символов текста, статус, иконка «изменено вручную»
- Значок ⚠️ если `needs_review = true`

**Редактирование рассылки:**
- Поле даты/времени отправки (редактируемо)
- Поле текста (редактируемо, textarea)
- Кнопка «Сбросить к авто-шаблону»
- Кнопка «Отправить сейчас»

**Авто-шаблоны:**

`pre_5min`:
```
⏰ Через 5 минут начинается выступление {speaker.name}!
Тема: {session.title}
Спикер подарит: {session.gift_description}
Подключайтесь: {conference.stream_url_day_N}
```

`post_thanks`:
```
👏 Спасибо {speaker.name} за выступление!
Тема была: {session.title}
Следите за программой в приложении.
```

`day_start`:
```
🌅 Доброе утро! Сегодня День {day} конференции iVision.
Программа дня:
{session_list_for_day}
```

`speaker_intro`:
```
{speaker.photo}
🎤 {speaker.name}
{speaker.title}, {speaker.company}

📌 Регалии:
{speaker.achievements}

Выступит с темой: «{session.title}»
Подарит участникам: {session.gift_description}

Telegram-канал: {speaker.telegram_url}
```

**Рассылки `speaker_intro`:** создаются в заданную дату для спикеров каждого дня, в порядке приоритета ролей (настраивается в разделе 6.3).

**Каскадное обновление:** при изменении сессии → авто-пересчёт не-отредактированных рассылок. Отредактированные → флаг `needs_review`.

### 6.5 Раздел «Услуги и Материалы»
- Список карточек с типом (`service` / `material`)
- Форма: название, описание, тип, признак платности (`is_paid`), URL ссылки действия (GetCourse или внешняя), `sort_order`
- `service` — в состояниях 2 и 3; `material` — только в состоянии 3

### 6.6 Раздел «Кодовые слова»
- Список кодов текущей конференции
- Форма: кодовое слово, кол-во выдаваемых билетов, привязка к спикеру (опционально)
- Коды хранятся в нижнем регистре; проверка при вводе — без учёта регистра

### 6.7 Раздел «Реферальная программа»
- Настройка призов: название, порог тикетов, описание
- Настройка начисления тикетов (за регистрацию, за реферала, за активность)

### 6.8 Раздел «Участники»
- Таблица: ник, имя, дата регистрации, тикеты, кол-во рефералов, флаг ветерана, флаг `has_filled_form`, флаг `channels_verified`
- Выгрузка в CSV/Excel

### 6.9 Быстрые действия (через Telegram-бота организатора)
- `/stats` — текущая статистика
- Уведомление при новой заявке спикера + кнопки «Одобрить» / «Отклонить»
- Уведомление если рассылка требует проверки (`needs_review`)

## 7. Интеграция с GetCourse

**Основной поток (регистрация):**
```
Участник нажимает «Зарегистрироваться» в TMA
→ Открывается GetCourse-лендинг в браузере
→ Участник заполняет форму (данные остаются на GetCourse, в TMA не переносятся)
→ GetCourse отправляет Webhook:
  POST /webhook/getcourse
  {
    telegram_id: "12345",
    email: "user@mail.com",
    first_name: "Анна",
    last_name: "Иванова",
    phone: "+79001234567",
    getcourse_id: "gc_789",
    ref_code: "ABC"
  }
→ Backend: сохраняет email/имя/телефон/getcourse_id в event_participants; статус interested → registered
→ TMA обновляется — открывается полный кабинет
→ Бот: приветственное сообщение + кнопка «Открыть приложение и пройти посвящение»
```

**Ветеранский путь:**
- При открытии TMA: если `is_veteran = true` — бот предлагает зарегистрироваться с предзаполненной формой
- URL на GetCourse передаётся с параметром `?tg_id=12345`, чтобы GetCourse мог идентифицировать пользователя
- После регистрации GetCourse присылает вебхук с `email`, `first_name`, `last_name`, `phone`, `getcourse_id` — сохраняются в TMA так же, как для новых участников
- После регистрации ветерана на новую конференцию: `channels_verified = false` (нужно заново подписаться на спикеров текущей конференции)
- Ветеран пропускает Level 0 анкету (`has_filled_form = true` уже), но обязан пройти «Клубный взнос»

**Level 0 / Квест (отдельная GetCourse-форма):**
```
Участник в [Игра] нажимает «Забрать» на карточке «Уровень 0: Свои люди»
→ Открывается отдельная GetCourse-анкета
→ GetCourse отправляет Webhook:
  POST /webhook/getcourse/form_completed
  { telegram_id: "12345", form_id: "quest_form" }
→ Backend: has_filled_form = true, is_in_community = true
→ Бот: ссылка на чат + гайд + Pop-up «Добро пожаловать в элиту»
```

**Реферальная цепочка:**
- Переход по реф. ссылке `?startapp=REF_CODE` → бот запоминает ref_code
- Webhook от GetCourse → ref_code привязывается к участнику → реферер получает +1 + тикет

## 8. Техническая архитектура

| Компонент | Назначение |
|---|---|
| `@ivision_conf_bot` | Telegram-бот, точка входа, уведомления |
| TMA (фронтенд) | React/Vue SPA, Telegram Web Apps API |
| Веб-админка | SPA/SSR на том же бэкенде |
| Бэкенд API | Node.js / Python, REST API, HMAC-валидация initData |
| БД | PostgreSQL (совместима с referalka) |
| Хранилище медиа | S3 / Cloudflare R2 |

**Telegram Web Apps API:**
- `initData` — авторизация без формы
- `startapp` — реферальный код
- `MainButton` — главный CTA
- `BackButton` — нативная кнопка назад
- `shareURL` — шеринг реф. ссылки
- `HapticFeedback` — вибрация при ключевых действиях
- `expand()` — полноэкранный режим
- CSS переменные `--tg-theme-*` — адаптация под тему пользователя

## 10. Масштабирование на других клиентов

При развёртывании для клиента referalka (по отдельному договору):
1. Деплоится та же кодовая база с новым ботом клиента
2. В таблицу `conferences` добавляется запись для новой конференции
3. Клиент получает доступ к своей веб-админке
4. Бот — собственный бот клиента (его актив, его аудитория)
5. Схема БД идентична — переносимость без переписывания
